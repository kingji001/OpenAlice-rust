# parity/

This directory holds the parity infrastructure for the Rust migration:
fixtures the Rust port must reproduce, generators that emit those
fixtures, and CLIs that drive the TS reference implementation.

See [`docs/RUST_MIGRATION_PLAN.v4.md`](../docs/RUST_MIGRATION_PLAN.v4.md)
for migration phasing. See [`context-worksheets/README.md`](context-worksheets/README.md)
for sub-PR coordination.

## Layout

| Path | Purpose | Lifecycle |
|---|---|---|
| `fixtures/operations/` | Per-Operation cases (≥200) for Phase 3 parity | Generated by `generators/operations.ts` |
| `fixtures/sentinels/` | UNSET_* in every position | Generated by `generators/sentinels.ts` |
| `fixtures/git-states/` | Captured GitExportState per scenario | Captured by `run-ts.ts --scenario=…` |
| `fixtures/scenarios/` | Hand-authored multi-step inputs | Hand-edited |
| `fixtures/legacy-paths/` | Saved states at legacy persister paths | Hand-authored / scenario-derived |
| `fixtures/orders-on-wire/` | `JSON.stringify(Order/Contract)` snapshots | Generated by `generators/orders-on-wire.ts` |
| `generators/` | Fixture-emitting scripts (deterministic, idempotent) | Run via `pnpm tsx parity/generators/<name>.ts` |
| `run-ts.ts` | TS reference CLI: drives TradingGit through lifecycle | Run via `pnpm tsx parity/run-ts.ts <fixture>` |
| `load-legacy.ts` | DoD command: verifies legacy-path snapshots load | Run via `pnpm tsx parity/load-legacy.ts` |
| `decimal-inventory.md` | FFI-boundary numeric-field audit driving Phase 1b | Hand-maintained |

## How to regenerate fixtures

```bash
pnpm tsx parity/generators/operations.ts       # → fixtures/operations/
pnpm tsx parity/generators/sentinels.ts        # → fixtures/sentinels/  (depends on inventory)
pnpm tsx parity/generators/orders-on-wire.ts   # → fixtures/orders-on-wire/  (depends on operations)
```

All generators are deterministic and idempotent — re-running produces
byte-identical output. CI asserts no diff after regen.

## How to run the TS reference

```bash
pnpm tsx parity/run-ts.ts parity/fixtures/operations/placeOrder/case-buy-mkt-day-001.json
pnpm tsx parity/run-ts.ts --scenario=parity/fixtures/scenarios/01-single-commit.scenario.json
pnpm tsx parity/run-ts.ts --all                 # batch over all operation fixtures
```

Phase 3 (Rust port) ships a sibling `run-rust` and CI diffs the two outputs.
